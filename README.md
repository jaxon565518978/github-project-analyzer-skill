# GitHub 开源项目分析器

分析本地克隆的 GitHub 仓库，产出全面的技术分析报告。

## 功能特性

- **整体技术框架分析**：识别架构风格、技术栈、通信模式
- **核心模块梳理**：自动提取模块边界、依赖关系、入度/出度
- **方案设计分析**：识别设计模式、数据流、抽象层、扩展性
- **创新点挖掘**：验证真实创新，排除 README 营销话术
- **代码质量评估**：测试比例、代码异味、注释率、圈复杂度
- **安全合规扫描**：敏感信息泄露、漏洞模式、许可证兼容性
- **性能特征分析**：并发策略、内存分配、优化手段、benchmark 覆盖
- **可视化输出**：自动生成 Mermaid 依赖图、API 接口清单
- **代码考古学**：通过 git 历史理解架构演进

## 前置条件

1. 将目标 GitHub 仓库克隆到本地：
   ```bash
   git clone https://github.com/owner/repo ~/projects/repo
   ```
2. 系统已安装 Python 3（用于运行分析脚本）

## 使用方法

将本地仓库路径提供给 QoderWork，例如：

> "帮我分析一下 ~/projects/cobra"

QoderWork 会自动执行以下流程：

1. 验证本地目录并读取仓库元数据
2. 运行 6 个自动化分析脚本获取量化数据
3. 通过架构反推五步法提炼设计意图
4. 提取核心模块并分析方案设计
5. 识别创新点与差异化设计
6. 评估代码质量、安全合规、性能特征
7. 生成 Mermaid 依赖图和 API 清单
8. （可选）代码考古学分析
9. 整合所有结果，产出结构化 Markdown 报告

## 分析脚本说明

| 脚本 | 功能 | 输出 |
|------|------|------|
| `analyze_deps.py` | 模块依赖分析：import 关系、循环依赖、入度/出度 | `deps.json` |
| `analyze_arch.py` | 架构元素提取：入口、接口、结构体、工厂函数 | `arch.json` |
| `analyze_quality.py` | 代码质量：测试比例、异味、注释率、复杂度 | `quality.json` |
| `analyze_security.py` | 安全合规：敏感信息、漏洞模式、许可证 | `security.json` |
| `analyze_performance.py` | 性能特征：并发模式、内存分配、优化手段 | `perf.json` |
| `generate_mermaid.py` | 可视化：Mermaid 依赖图、API 接口清单 | `.mmd` / `api.json` |

所有脚本支持 Go / Python / Rust / Java / TypeScript / JavaScript / C / C++ 等主流语言。

## 报告模板

根据项目类型自动选择报告模板，避免空章节：

| 项目类型 | 模板文件 | 侧重维度 |
|----------|----------|----------|
| 库 / SDK | `REPORT_TEMPLATE_LIB.md` | API 设计、扩展性、兼容性、使用示例 |
| 服务 / 后端 | `REPORT_TEMPLATE_SERVICE.md` | 架构、部署、安全、数据流、性能伸缩 |
| CLI / 工具 | `REPORT_TEMPLATE_CLI.md` | 命令设计、配置系统、使用体验、Shell 集成 |
| 通用 | `REPORT_TEMPLATE.md` | 标准全维度分析 |

类型自动检测基于代码特征：是否存在 main 入口、HTTP handler、Dockerfile、`package.json` 中的 `bin`/`main` 字段等。

## 报告结构（以通用模板为例）

1. 项目概览
2. 技术栈
3. 架构概览（含 Mermaid 依赖图）
4. 核心模块（含方案设计分析）
5. 创新点
6. 代码质量评估（量化指标）
7. 安全与合规（量化指标）
8. 性能特征（量化指标）
9. API 与接口清单
10. 开发与社区
11. 总结与收获

## 输出定制

可根据需求调整分析深度：

- **快速概览**：跳过深度分析，基于扫描结果输出 1 页摘要
- **深度分析**：扩展架构反推和代码走读，增加 UML 描述
- **安全审查**：聚焦威胁模型与安全设计分析
- **性能剖析**：结合 benchmark 结果做热点分析
- **PPT 演示**：调用 pptx skill 生成幻灯片
- **表格汇总**：调用 xlsx skill 生成数据表格

## 并行分析

对于模块数超过 10 个或代码量超过 10 万行的大型项目，支持分模块并行分析：

1. 先完成全局扫描和架构反推
2. 将核心模块拆分为 3–5 个批次
3. 并行派生子 agent 分析各批次
4. 汇总整合，检查模块间一致性

## 注意事项

- 本 skill **不执行** `git clone`，需要用户提前准备好本地代码
- 如果用户只提供了 GitHub URL，会提示用户先克隆代码
- 超大仓库的分析脚本运行时间可能较长，可通过 `--max-nodes` 等参数限制输出规模
- 安全扫描结果中的敏感信息可能是测试数据或示例值，需人工二次确认
